SentinelGuard 网页恶意检测运行说明

一、项目范围

当前版本只保留“网页恶意检测”能力，支持输入完整 URL 或域名。
不再包含小程序、应用软件、爬虫、多引擎报告等旧功能。

当前检测流程采用“**静态检测 → 可选深度研判**”的两段式设计：

1. 先做网页静态恶意分析，提取 URL 结构、跳转链、页面 HTML/DOM 摘要、可见文本、表单/下载行为与风险证据；
2. 如果开启深度研判，则在静态结果基础上继续进行**多专家、多模型协同分析**，生成更细的处置建议、专家观点和主持人最终总结。

也就是说：**深度研判不是替代静态检测，而是在静态检测结果上继续增强**。

简单理解：

- **静态检测报告**：偏“事实与证据”，更关注 URL 结构、页面跳转、HTML/DOM 摘要、表单/下载行为、可疑关键词、风险规则命中等可验证内容；
- **深度研判报告**：偏“专家会诊”，在静态证据基础上，由不同角色分别输出结论，最后由主持人统一收敛并给出最终建议。

> 说明：当前版本不生成网页截图；如果需要观察“页面长什么样”，请查看静态检测中提取的 HTML 摘要、可见文本摘要、表单/脚本信息和跳转链，它们能够替代截图完成大部分页面界面研判。

因此，静态报告适合快速确认“有没有问题”，深度研判适合进一步回答“为什么可疑、像什么攻击、该怎么处置”。

二、运行环境

- Python 3.11+ 推荐
- Windows 10/11、Linux、macOS 均可
- 需要联网访问目标网址

三、安装依赖

进入项目目录后执行：

```bash
pip install -r requirements.txt
```

如果你已经有虚拟环境，先激活环境再安装依赖。

四、可选配置

复制 `.env.example` 为 `.env`，按需修改：

- `HOST`：主机地址，默认 `0.0.0.0`
- `PORT`：端口，默认 `5000`
- `DETECTION_REPORT_DIR`：报告输出目录，默认 `sentinel_reports`
- `DETECTION_TIMEOUT_SECONDS`：页面抓取超时时间，默认 `6`
- `DETECTION_USER_AGENT`：抓取时的 User-Agent
- `ENABLE_DEEP_ANALYSIS`：是否默认启用大模型深度研判
- `DETECTION_HOST_*`：主持人角色配置，负责统筹总结各专家意见
- `DETECTION_STATIC_*`：静态分析员角色配置，负责规则推理、证据归纳
- `DETECTION_BEHAVIOR_*`：行为分析员角色配置，负责跳转链、交互行为、下载/表单分析
- `DETECTION_INTEL_*`：情报分析员角色配置，负责背景情报与威胁归因
- `DETECTION_ADVICE_*`：处置建议员角色配置，负责输出可执行处置建议
- `DETECTION_HTTP_PROXY` / `DETECTION_HTTPS_PROXY` / `DETECTION_ALL_PROXY`：访问目标网站或模型接口需要代理/VPN 时使用

建议的角色模型搭配如下：

- **主持人**：`qwen3.5-plus` 或 `qwen-plus`，适合中文统筹和会诊收敛
- **静态分析员**：`deepseek-reasoner`，适合规则推演和证据归纳
- **行为分析员**：`gemini-2.5-pro`，适合长上下文页面行为、HTML 线索和跳转链分析
- **情报分析员**：`gpt-5`，适合跨源知识整合和威胁背景归因
- **处置建议员**：`gemini-2.5-flash`，适合快速生成可执行建议

如果你的目标网址在国内网络下无法访问，或者模型服务本身需要外网通道，请先配置代理再运行，否则抓取页面时可能只能拿到不完整的响应，甚至完全失败。

代理示例：

```env
DETECTION_HTTP_PROXY=http://127.0.0.1:7890
DETECTION_HTTPS_PROXY=http://127.0.0.1:7890
DETECTION_ALL_PROXY=socks5://127.0.0.1:1080
```

### 配置优先级说明

本项目在实际分析时，会优先使用**网页前端输入**的运行时配置，其次才读取 `.env`：

1. **前端填写的 LLM API Key / Base URL**：优先用于深度研判中的大模型调用；
2. **前端填写的代理配置**：优先用于网页抓取以及深度研判的大模型请求；
3. **`.env` / 系统环境变量**：仅在前端未提供对应参数时作为默认值。

也就是说，如果你在网页前端输入了 `llm_api_key`、`llm_base_url`、`proxy_http`、`proxy_https` 或 `proxy_all`，实际分析会优先使用这些值，而不是 `.env` 里的配置。

> 备注：如果你使用的是本地/内网代理，`http://127.0.0.1:7890` 这种写法通常是对的；`https://127.0.0.1:7890` 往往会导致代理握手失败。

如果你暂时不做深度研判，可以保持：

```env
ENABLE_DEEP_ANALYSIS=False
```

> 提示：如果不配置深度研判三项参数，建议把 `ENABLE_DEEP_ANALYSIS` 设为 `False`，只做静态检测。

五、启动方式

当前项目的主入口是 `app.py`。

1. 最简单的运行方式：

```bash
python app.py https://example.com
```

2. **只做结构检测，不抓取页面 HTML**：

```bash
python app.py https://example.com --no-fetch
```

3. 指定输出目录：

```bash
python app.py https://example.com --output-dir sentinel_reports
```

4. 开启大模型深度研判：

```bash
python app.py https://example.com --deep
```

注意：深度研判需要在 `.env` 中至少配置主持人/专家角色对应的一组模型参数；如果你希望快速跑通，也可以先只配置主持人和静态分析员，再逐步补齐其余专家角色。

如果你希望默认就启用深度研判，也可以直接在 `.env` 中设置：

```env
ENABLE_DEEP_ANALYSIS=True
```

这样执行 `python app.py 网址` 时，会先跑静态检测，再自动进入深度研判。

六、运行结果

运行后会在终端输出：

- 风险等级
- 风险分数
- 分析模式
- HTML 报告路径
- Markdown 报告路径
- 证据摘要
- 深度研判时还会输出各专家使用的模型映射，以及主持人最终总结

其中：

- **静态检测** 会给出基础风险判断、URL/页面 HTML 摘要、可见文本摘要、跳转链和风险规则证据；
- **深度研判** 会在静态结论基础上，补充主持人、静态分析员、行为分析员、情报分析员、处置建议员的协同意见，并在报告中记录每个角色实际使用的模型；
- **主持人总结** 会把各专家建议做一次统一整理，输出最终结论和处置建议，便于人工复核和后续处置。

默认报告保存在：

```text
sentinel_reports/
```

七、测试与诊断脚本

项目里提供了两个用于诊断网络问题的脚本，建议优先使用下面这个：

### 1）推荐：网络连通性诊断脚本

路径：

```text
utils/network_diagnose.py
```

作用：

- 测试**直连**是否能访问外网
- 测试**代理/VPN**是否可用
- 判断是**VPN/代理问题**还是**代码请求逻辑问题**

运行方式：

```bash
python utils/network_diagnose.py https://example.com
```

如果想输出 JSON 结果：

```bash
python utils/network_diagnose.py https://example.com --json
```

如果你想手动指定代理：

```bash
python utils/network_diagnose.py https://example.com --proxy-http http://127.0.0.1:7890 --proxy-https http://127.0.0.1:7890
```

判断结果时可以参考：

- **直连成功，代理失败**：更可能是 VPN / 代理节点或代理配置有问题；
- **直连失败，代理成功**：说明只能走代理访问，直连网络不可用；
- **直连与代理都失败**：更可能是外网出口、目标站点或本机网络有问题；
- **诊断脚本可访问，但网页分析仍失败**：更可能是业务代码、请求头、超时、证书或目标站点反爬问题。

### 2）补充：项目内抓取逻辑对比脚本

路径：

```text
tests/vpn_network_diagnosis.py
```

作用：

- 同时对比 `requests` 原始请求与项目内 `url_analyzer` 抓取逻辑
- 适合排查“代理能上网，但项目代码还是抓不到”的情况

运行方式：

```bash
python tests/vpn_network_diagnosis.py --target https://example.com
```

只输出 JSON：

```bash
python tests/vpn_network_diagnosis.py --target https://example.com --json
```

手动指定代理：

```bash
python tests/vpn_network_diagnosis.py --target https://example.com --proxy-http http://127.0.0.1:7890 --proxy-https http://127.0.0.1:7890
```

如果这个脚本里“代理探测成功”，但项目内抓取失败，那就优先怀疑：

- 代理参数没有正确传入前端表单
- 代理协议写错了（例如把 HTTP 代理写成了 `https://...`）
- 目标站点对当前 User-Agent / TLS / 超时策略有限制
- 代码中的抓取分支与代理分支不一致

### 3）页面界面观察方式说明

当前版本不再做网页截图，因此观察页面界面时，建议重点看以下信息：

- **HTML 摘要**：页面标题、主要文本、关键标签与布局线索；
- **DOM 线索**：表单数量、隐藏字段、脚本外链、跳转指令；
- **可见文本**：登录提示、支付诱导、验证码、品牌仿冒词；
- **跳转链**：是否多级跳转、是否跨域、最终是否降级到 HTTP；
- **下载行为**：是否出现 APK、脚本、压缩包等可疑下载入口。

这些信息足以替代大多数“看截图”的初筛需求，并且更容易做规则化、结构化研判。

七、报告文件

每次运行会生成两份文件：

- `.html`：可直接用浏览器打开
- `.md`：可用于文本查看或二次整理

深度研判完成后，报告内容通常会比静态报告多出以下部分：

- 专家模型映射：每个角色使用了哪个模型
- 专家观点汇总：各角色分别给出的结论和建议
- 主持人最终总结：对专家意见的统一收敛结果

当前报告 UI 已做了优化，具备以下特点：

- 顶部概览卡片，便于快速看风险等级和风险分数；
- 分区展示“统一 IR、跳转链、风险证据、协同研判、扩展信息”；
- 风险证据和页面摘要支持滚动查看，避免内容过长时页面过于臃肿；
- HTML 报告支持更清晰的视觉层次，适合直接浏览和复核；
- Markdown 报告保留结构化小标题，便于复制、归档或二次整理。

八、建议的使用流程

1. 准备一个可疑网址
2. 执行 `python app.py 网址`
3. 先看静态检测的风险等级、跳转链和页面摘要
4. 如果启用了深度研判，再看协同意见与处置建议
5. 如果目标站点需要代理/VPN，请确认 `.env` 中的代理已正确配置
6. 打开 `sentinel_reports` 中生成的 HTML 报告
7. 根据证据链做人工复核

九、示例

```bash
python app.py http://example.com/login?redirect=http://evil.test
```

apk:


1. 入口识别
    - 页面上传 .apk 文件后，后端把它保存到临时路径。
    - parse_target(..., target_type="apk") 会把它识别成 TargetIR(target_type="apk")，并挂上 APKIR。
  2. APK 解析
    - SentinelGuard/analyzers/apk_analyzer.py 会把 APK 当作 zip 包读取。
    - 解析这些信息：
        - 文件大小
      - SHA256
      - AndroidManifest.xml
      - 包名、版本号
      - 权限声明
      - activity / service / receiver / provider
      - 一些文本字符串线索
      - META-INF 签名相关文件
  3. 规则打分
    - 根据提取到的特征生成 DetectionFinding：
        - 敏感权限过多
      - 缺少包名
      - 可疑字符串
      - 可疑组件
      - 文件体积异常大
      - 签名信息需要复核
    - 如果没命中明显规则，也会给一个低危基线结论。
  4. 报告生成
    - judgement.py 把 APK 结果封装成 DetectionReport
    - report.py 再把结果渲染成 HTML / Markdown
  5. 页面展示
    - 页面会显示：
        - APK 文件名
      - 路径
      - 包名
      - SHA256
      - 证据列表
      - 研判结论
      - 下载报告
