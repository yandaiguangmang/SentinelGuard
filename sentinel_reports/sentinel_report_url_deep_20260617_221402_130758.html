按严重度与规则优先级呈现

HIGH（2）

页面强制将 HTTPS 降级为 HTTP HIGH

规则：DEEP_STATIC_FORCED_HTTP_DOWNGRADE

页面通过 JavaScript 代码和 `<noscript>` 标签内的 Meta Refresh 指令，明确指示浏览器将当前 HTTPS 连接降级为 HTTP 连接。这直接削弱了传输安全性，可能导致敏感信息泄露或被篡改。

证据JavaScript: `location.replace(location.href.replace("https://","http://"));`；Meta Refresh: `0;url=http://www.baidu.com/` 建议警惕任何强制降级 HTTPS 的行为，即使目标域名看似合法。此类行为是典型的中间人攻击前置步骤，建议阻止访问或在严格隔离环境中进行复核。

页面执行HTTPS降级攻击 HIGH

规则：DEEP_BEHAVIOR_HTTPS_DOWNGRADE

页面内嵌的JavaScript代码尝试将URL协议从 'https://' 替换为 'http://' 并立即跳转，同时在<noscript>标签内设置了Meta Refresh作为后备方案，同样指向HTTP地址。此行为旨在剥离TLS加密，使用户暴露在流量监听和篡改的风险之下。

证据JavaScript: location.replace(location.href.replace("https://","http://")); Meta Refresh: <meta http-equiv="refresh" content="0;url=http://www.baidu.com/"> 建议立即阻断对该URL的访问。HTTPS降级是明确的恶意指标，不应信任任何由该页面发起或跳转到的后续页面。

MEDIUM（5）

页面包含自动跳转指令 MEDIUM

规则：PAGE_META_REFRESH

Meta refresh 可在用户无感知情况下跳转到其他页面。

证据0;url=http://www.baidu.com/ 建议关注浏览器最终地址栏域名是否发生变化。

页面使用多种跳转机制 MEDIUM

规则：DEEP_STATIC_MULTIPLE_REDIRECT_MECHANISMS

页面同时使用了客户端 JavaScript (`location.replace`) 和 HTML `<noscript>` 标签内的 `meta refresh` 两种不同的跳转机制。这种多重机制可能旨在确保在不同浏览器环境（例如，JavaScript 被禁用时）或规避某些安全检测时，跳转行为仍能成功执行。

证据JavaScript `location.replace` 和 `<noscript><meta http-equiv="refresh"></noscript>` 建议分析多种跳转机制的组合使用，尤其关注其跳转目标是否一致以及是否存在混淆或规避检测的意图。在动态分析中验证所有可能的跳转路径。

代理/链路环境差异可能导致内容改写伪像 MEDIUM

规则：DEEP_INTEL_ENV_PROXY_VARIANCE

静态与浏览器抓取均显示通过企业代理访问，返回的 HTML 含 HTTPS→HTTP 降级指令，这与目标常规行为不符，提示可能存在前向代理/中间盒插入或准入页改写。

证据proxy_used=true；proxy=10.250.167.176:7890；HTML 含 location.replace("https://","http://") 建议在无代理直连与不同外网/VPN 出口（境内/境外、不同 ASN）重复抓取并比对：DNS 应答、TLS 证书链/CT、一跳与完整重定向链、响应差异。对可疑代理链路进行 HAR/PCAP 采集与证书指纹核验。

存在 HTTPS 降级重定向意图 MEDIUM

规则：DEEP_INTEL_DOWNGRADE_INTENT

页面内联脚本与 noscript meta 同时指向 http://www.baidu.com/，属于降级重定向模式，若为真实来源将削弱传输机密性与完整性。

证据JS: location.replace(location.href.replace("https://","http://"));；Meta: 0;url=http://www.baidu.com/；浏览器观察：最终为 https://www.baidu.com/ 建议核验目标在不同出口下的 HSTS（是否预加载/头部 max-age）、证书链与 SAN；确认正常路径是否保持全站 HTTPS。若仅在特定链路出现降级，优先排查链路改写。

页面存在客户端自动跳转逻辑 MEDIUM

规则：DEEP_ADVICE_CLIENT_SIDE_REDIRECT

页面HTML中包含JavaScript代码 `location.replace(location.href.replace("https://","http://"));` 和 `<noscript>` 标签内的 `<meta http-equiv="refresh" content="0;url=http://www.baidu.com/">`，尝试将当前HTTPS连接重定向至HTTP连接。尽管浏览器最终停留在原始HTTPS地址，但此类行为可能被恶意利用进行协议降级或重定向至其他恶意站点。

证据HTML摘要显示存在 `location.replace` 脚本和 `meta http-equiv="refresh"` 标签，均指向 `http://www.baidu.com/`。 建议不建议直接拦截该页面，因为其最终停留在原始HTTPS地址。但建议对该页面进行隔离访问，以防止潜在的协议降级风险。同时，强烈建议进行沙箱复核，以验证客户端脚本和Meta Refresh的实际执行效果和最终跳转目标。此事件应进行留痕，以便后续审计和分析。

外网/VPN 节点访问差异的研判要点 LOW

规则：DEEP_INTEL_VPN_ACCESS_DIFFERENCES

当目标依赖外网/VPN 访问时，DNS 解析、CDN 边缘、Geo/合规策略与证书链可能随出口节点改变，从而影响重定向、内容与安全标头呈现。

环境元数据不一致可能导致误判 LOW
