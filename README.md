# SentinelGuard 哨塔

> 面向小程序、应用软件与网址的协同恶意检测平台

SentinelGuard 哨塔是一个轻量级安全研判平台，聚焦三类高频风险入口：**URL、APK、小程序（预留）**。项目将统一输入、统一解析、静态分析、协同研判与结构化报告整合在同一条检测链路中，适合用于安全展示、样本初筛与研究演示。

## ✨ 特性

- **URL 检测**：支持域名结构、跳转链、页面 HTML/DOM 摘要、下载链接、敏感关键词等分析
- **APK 检测**：支持 Manifest、权限、组件、签名线索、字符串与文件特征分析
- **协同研判**：通过主持人、静态分析员、行为分析员、情报分析员、处置建议员形成多角色结论
- **报告输出**：自动生成 HTML / Markdown 报告，便于查看、归档与复核
- **可扩展架构**：为小程序、动态分析、威胁情报联动等能力预留扩展空间

## 🧩 支持对象

### URL
- 自动补全协议
- 结构与参数风险识别
- 页面 HTML 抓取、DOM/文本摘要与线索提取
- 跳转链分析
- 深度协同研判

> 说明：当前版本不再生成网页截图，而是通过页面 HTML、DOM 结构、可见文本、表单、脚本外链和跳转链来观察页面界面与行为线索。

### APK
- 文件大小与 SHA256
- AndroidManifest 解析
- 权限与组件分析
- 签名线索与字符串分析
- 风险规则打分

### 小程序
- 当前为预留方向，后续可扩展页面结构、授权请求与外链分析

## 🏗️ 架构概览

1. **输入层**：接收 URL / APK / 小程序对象
2. **解析层**：统一转为 IR（中间表示）
3. **分析层**：执行静态规则检测与线索提取
4. **协同研判层**：多角色联合输出结论
5. **报告层**：生成 HTML / Markdown 结构化报告

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动服务

```bash
python app.py
```

默认访问地址：`http://127.0.0.1:5000`

### 3. 使用方式

- **URL 检测**：在首页输入网址，选择 URL 检测后开始分析
- **APK 检测**：切换到 APK 检测，上传 `.apk` 文件或填写本地路径后开始分析

## 📄 报告输出

检测完成后会自动生成：

- HTML 报告
- Markdown 报告

报告中包含：

- 风险等级
- 风险分数
- 目标信息
- 风险证据
- 协同研判结论
- 处置建议

## ⚙️ 配置说明

主要配置位于 `.env` 或环境变量中：

- `HOST`
- `PORT`
- `DETECTION_REPORT_DIR`
- `DETECTION_TIMEOUT_SECONDS`
- `DETECTION_USER_AGENT`
- `ENABLE_DEEP_ANALYSIS`

如果启用深度研判，还可以配置不同角色的模型服务地址与 API Key。

## 🔎 APK 检测逻辑

当前 APK 检测采用**静态分析**方式，主要关注：

- 文件大小与哈希
- Manifest 信息
- 权限列表
- 组件列表
- 签名线索
- 可疑字符串

后续可继续扩展：

- 反编译与代码级分析
- 动态沙箱行为监控
- 域名 / IP 威胁情报联动
- 样本家族聚类与相似度分析
